# ADR 0001：第一阶段采用 WDL 1.0 Compatibility Profile

- 状态：Accepted
- 日期：2026-07-23
- 决策范围：Phase 1

## Context

BioWorkflowManage 的内部模型需要面向长期演进，但第一阶段必须选择一个明确、可验证的 WDL 输出目标。若同时支持多个 WDL 版本和不同执行器细节，Renderer、测试矩阵和错误处理都会显著扩大，影响 Graph -> WDL 核心预演。

## Decision

第一阶段：

- Workflow Graph target 固定 `language = wdl`；
- 默认 `version = 1.0`；
- 主 profile 为 `cromwell-compatible`；
- 可保留 `miniwdl-compatible` profile，但不能改变 Graph 核心语义；
- miniwdl 用于独立静态校验；
- WOMtool 作为推荐第二校验器；
- Compiler IR 与 WDL 版本解耦；
- 未来通过新增 Renderer 支持其他 WDL 版本，不修改既有 Graph 含义。

## Consequences

正面：

- 第一阶段测试目标明确；
- golden WDL 稳定；
- 兼容性问题集中在 Renderer/Profile；
- 不需要让 ToolSpec 保存 WDL 版本语法；
- 后续新增版本不会推翻 Graph 和 ToolSpec。

代价：

- 第一阶段不能使用更高版本 WDL 的全部语法；
- 某些执行器专属 runtime 字段只能记录在 manifest；
- 多版本支持需要额外 Renderer 与 golden test。

## Guardrails

- 不允许在代码中根据环境变量静默更改 WDL version；
- target/profile 必须写入 Graph、IR 和 manifest；
- Renderer 输出变化必须更新 golden files；
- 不支持的 target 返回 `WG019` 或 `RD001`；
- ToolSpec 不得包含完整 WDL task 文本。
